原理解释 #

首先，Babel的原理是简单的，例如这样的ES6代码：

const hello = () => {
    console.log('Hello, babel!')
}

最终会被编译为：


var hello = function() {
    console.log('Hello, babel!')
}

上述过程中，Babel需要将代码中的const声明全部转换为var，还需要将ES6中特有的箭头函数翻译为普通函数声明。

为了完成这件事，并使其足够的可扩展，Babel将上述的两件工作抽象成了两个插件：

@babel/plugin-transform-block-scoping@babel/plugin-transform-arrow-functions

在Babel工作时，这两个插件会分别完成自己各自的工作，完成代码的翻译。

因此，插件是Babel的核心。

如果一个插件都不使用，Babel不会对源码进行任何的处理，输入的代码会原样地进行输出。

一、下载依赖包 #

npm i -D @babel/core @babel/cli

@babel/core是Babel的核心，它的作用是解析代码和转换代码。 @babel/cli，见名知意，它是Babel的命令行，这里使用它只是为了不用手动去写js代码来完成代码的解析和转换。

二、添加配置文件 #

在根目录下新建babel.config.json文件：

{
  "plugins": [
    "@babel/plugin-transform-block-scoping",
    "@babel/plugin-transform-arrow-functions",
    "@babel/plugin-transform-classes"
  ]
}

这里我们不需要额外的配置，就配置3个插件，用来转换我们的代码即可。

注：以上三个插件都需要安装到开发依赖中。

三、写点代码 #

在src下新建index.js：

const hello = () => {
  console.log('hello, babel!')
}

class Person {
  constructor(name) {
    this.name = name
  }

  sayHello() {
    hello()
  }
}

以上代码我们使用了ES6的三个特性：局部作用域、箭头函数和类。

四、调用命令，转换代码 #

在控制台中输入命令：

npx babel src --out-dir dist

命令的含义非常简单，将src目录下的文件转换后放置在dist目录下。

六、撤掉一个插件试试？ #

在配置文件中，我们去除@babel/plugin-transform-classes，只保留局部作用于和箭头函数的转换插件，看看生成的会是什么效果呢？

var hello = function () {
  console.log('hello, babel!');
};

class Person {
  constructor(name) {
    this.name = name;
  }

  sayHello() {
    hello();
  }

}

果然，撤掉ES6类的支持后，Babel没有为我们提供任何的代码转换，而是将代码原样地输出了出来。
